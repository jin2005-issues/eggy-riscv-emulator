# rv32ima riscv emulator in lua running on eggy party
可以在蛋仔派对运行Linux的rv32模拟器，完整运行需要配合相关界面和启动蛋码

## 模拟器相比于原项目做了如下主要改动
对蛋仔lua5.4运行环境做了一定适配（位运算、限制浮点数出现、分帧运行）

对指令解码结果进行缓存，在非luajit环境性能提升约一倍

## Linux相关文件加载方式

蛋仔lua限制了io文件读取以及lua代码、蛋码、游戏表格容量大小。因此需要将Linux相关二进制文件通过zstd+base85压缩并转换为文本形式后，进行分割，分别存放于有限容量的lua文件、蛋码和表格中，运行时再进行合并。

经测试，lua文件、蛋码、游戏表格容量都为600k左右

第一部分以lua代码形式存放在Image_part1.lua文件中

第二部分位于Image_text文件夹中，分12份，每份再分为不超过30000字节的文本块，存放于蛋码和游戏表格中

## 与蛋码的对接

UART输入： 通过keyboard.lua接受界面按键事件，转换为相应终端输入后，发送到UART输入缓冲区

UART输出：vt100.lua实现了一个终端模拟器，uart输出字符串经终端模拟器处理后打印到界面文本框

启动：蛋码调用startCore()接口函数后，模拟器开始加载Linux并设置运行标志开始分帧运行
